stork-m     移动端页面<br />
stork       SpringBoot后端<br /><br />
对于水站的生态检测，硬件周期性检测各个水站的流速，电导率，溶解氧，氨氮，总磷等指标，采用Modbus串行协议完成硬件检测数据上传至服务器，后端根据指标读取评价水源质量。前端部分分为公众页面，后台管理，移动端三部分，公众页面用原生js完成了河道水资源生态情况通知公告以及水站表扬惩罚，提供大众监督和建议功能。后台管理采用Vue-cli脚手架搭建，用Element组件库实现页面的，用Vuex管理一些用户登录，角色，错误日志，历史浏览之类的状态。后台管理可以查看实时和历史水站数据，报警记录，预测趋势（一期仅直接加减法预测，二期加入算法实现），系统用户登录，权限管理，检测送样任务派发查看进度，检测机构，辖区管理。移动端也是用Vue-cli，用内联框架封装成App的，主要是河长，送样人员，检测人，巡查员用来查看完成任务的。此外用Echarts完成数据展示。都是采用axios实现前后端的数据交互，后端采用Spring Boot框架，用Nginx解决跨域。
